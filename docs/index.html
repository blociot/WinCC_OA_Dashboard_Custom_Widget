Overview

The Custom Chart widget adds a programmable charting component to the WinCC OA Dashboard. Instead of being limited to pre-built chart types, you write JavaScript code that returns an Apache ECharts option object. This gives you access to the full ECharts library: line charts, bar charts, gauges, heatmaps, pie charts, scatter plots, and dozens more.

Key Features

  • Full ECharts API — any chart type ECharts supports, this widget supports
  • Up to 5 live datapoints — connect WinCC OA datapoints and use their live values
  • Client-side history buffer — automatically accumulates live values over a configurable time range for trend charts
  • Real-time updates — chart refreshes instantly when datapoint values change via WebSocket
  • Error overlay — JavaScript errors are shown directly on the widget for easy debugging
  • Integrated code editor — edit chart code directly in the widget settings panel

Option A: Fresh Project (no existing customizations)

1

Copy the data/ directory from this repository into your WinCC OA project root:

xcopy /E /I "source\data" "YOUR_PROJECT\data"
2

Restart the WinCC OA web server.

3

Open the Dashboard in your browser. Custom Chart should appear under Standard Library > Charts in the widget picker.

Option B: Project with existing index.html customizations

1

Copy the widget files (but NOT index.html):

xcopy /E /I "source\data\WebUI" "YOUR_PROJECT\data\WebUI"
xcopy /I "source\data\dashboard-wc\entry\custom-widgets.js" "YOUR_PROJECT\data\dashboard-wc\entry\"
2

Open your project's data/dashboard-wc/index.html and add this entry to the imports object inside the <script type="importmap"> block:

"@etm-professional-control/wui-widgets/wui-widget-custom-chart/wui-widget-custom-chart.js": "./entry/custom-widgets.js"
3

Restart the WinCC OA web server and hard-refresh the browser (Ctrl+Shift+R).

Note: If your project does not yet have a custom index.html, first copy the installation default: 
copy "C:\Program Files\Siemens\WinCC_OA\3.21\data\dashboard-wc\index.html" "YOUR_PROJECT\data\dashboard-wc\index.html"
Then add the import map entry.

Usage

Adding the Widget to a Dashboard

  1. Open the Dashboard and create or edit a dashboard page
  2. Click Add Widget
  3. Expand Standard Library > Charts
  4. Drag Custom Chart onto the grid
  5. A default line chart will appear

Configuring the Widget

Click the widget's edit (gear) icon to open the settings sidebar. You will see these sections:

Content Section (always open)

  • Datapoint 1 — Select the primary WinCC OA datapoint to monitor
  • Time Range — How long to keep values in the history buffer (e.g. 30s, 5m, 1h, 1d)
  • Chart Code — JavaScript code editor where you write the ECharts configuration

Additional Datapoints (collapsed by default)

  • Datapoint 2 through Datapoint 5 — Optional additional datapoints for multi-series charts

Writing Chart Code

The Chart Code editor expects JavaScript that returns an ECharts option object. Your code has access to live datapoint values and accumulated history.

Tip: You can browse the ECharts Examples Gallery, find a chart you like, and adapt its code for this widget. See the Converting ECharts Examples section below.

API Reference

Your chart code runs as a JavaScript function with the following variables available:

Variable Type Description
dp[0] .. dp[4] Object Array of datapoint objects (up to 5). Always 5 entries; unused ones have value: null.
dp[i].value any The latest live value received via WebSocket. Can be number, string, boolean, etc.
dp[i].name string The datapoint address (e.g. System1:ExampleDP_Arg1.)
dp[i].history.values Array Array of past values, accumulated client-side within the configured time range
dp[i].history.timestamps Array Array of Unix timestamps (milliseconds) corresponding to each value
animation boolean Dashboard animation setting
font string Dashboard font family
bgColor string Dashboard background color

Time Range Format

ValueDuration
30s30 seconds
5m5 minutes
1h1 hour (default)
12h12 hours
1d1 day

How the History Buffer Works

Each time a datapoint value changes (via WebSocket), the widget stores the value and current timestamp into a client-side ring buffer. Values older than the configured Time Range are automatically removed. This means:

  • History only accumulates while the dashboard page is open
  • Refreshing the page clears the buffer
  • The buffer grows as values arrive — it starts empty and fills over time
  • Use dp[i].history.timestamps and dp[i].history.values for trend/line charts
  • Use dp[i].value for current-value displays like gauges or bar charts

Examples

Ready-to-use chart code for the Custom Chart widget. Each example shows a live preview rendered with sample data. Copy any example into the Chart Code editor.

Basic Line Chart

1 DP
Simple line chart showing one datapoint's value over time.
Live Preview
var data = [];
var d = dp[0];
if (d.history && d.history.timestamps) {
  data = d.history.timestamps.map(function(t, i) {
    return [t, d.history.values[i]];
  });
}

return {
  animation: animation,
  textStyle: { fontFamily: font },
  backgroundColor: bgColor,
  tooltip: { trigger: 'axis' },
  xAxis: { type: 'time' },
  yAxis: { type: 'value' },
  series: [{
    name: d.name || 'Value',
    type: 'line',
    data: data,
    smooth: false
  }]
};

Multi-Line Chart

3 DPs
Multiple datapoints as separate lines on the same chart. Assign 3 datapoints in widget settings. ECharts original
Live Preview
var series = [];

for (var i = 0; i < 3; i++) {
  var d = dp[i];
  if (d.value == null && d.history.values.length === 0) continue;

  var data = d.history.timestamps.map(function(t, j) {
    return [t, d.history.values[j]];
  });

  series.push({
    name: d.name || ('DP ' + (i + 1)),
    type: 'line',
    data: data,
    smooth: true
  });
}

return {
  animation: animation,
  textStyle: { fontFamily: font },
  backgroundColor: bgColor,
  tooltip: { trigger: 'axis' },
  legend: { data: series.map(function(s) { return s.name; }) },
  xAxis: { type: 'time' },
  yAxis: { type: 'value' },
  series: series
};

Converting ECharts Examples

You can use any example from the Apache ECharts Examples Gallery in the Custom Chart widget. Here's how to convert them:

Step-by-Step Conversion

1. Replace option = { ... } with return { ... }

ECharts examples assign to a variable. The Custom Chart widget expects you to return the object.

// BEFORE (ECharts example)
option = {
  xAxis: { type: 'category', data: ['Mon', 'Tue'] },
  series: [{ data: [150, 230], type: 'line' }]
};

// AFTER (Custom Chart widget)
return {
  xAxis: { type: 'category', data: ['Mon', 'Tue'] },
  series: [{ data: [150, 230], type: 'line' }]
};

2. Replace static data with live datapoints

Instead of hardcoded arrays, use dp[i].value for current values or dp[i].history for time-series data. 

// BEFORE: static data
series: [{ data: [150, 230, 224, 218, 135], type: 'line' }]

// AFTER: live history from datapoint 1
var data = dp[0].history.timestamps.map(function(t, i) {
  return [t, dp[0].history.values[i]];
});
series: [{ data: data, type: 'line' }]

3. Use type: 'time' for the X-axis

When plotting live data over time, change the xAxis from category to time. Remove any static xAxis.data array — timestamps come from the data points. 

// BEFORE: category axis with labels
xAxis: { type: 'category', data: ['Mon', 'Tue', 'Wed'] }

// AFTER: automatic time axis
xAxis: { type: 'time' }

4. Add dashboard integration variables

Use animation, font, and bgColor so the chart respects dashboard settings.

return {
  animation: animation,
  textStyle: { fontFamily: font },
  backgroundColor: bgColor,
  // ... rest of options
};

5. Handle multiple series with a loop

If the original example has multiple static series, convert them to a loop over dp[]:

// BEFORE: multiple hardcoded series
series: [
  { name: 'Email', data: [120, 132, 101] },
  { name: 'Video', data: [220, 182, 191] }
]

// AFTER: dynamic from datapoints
var series = [];
for (var i = 0; i < 3; i++) {
  var d = dp[i];
  if (d.value == null && d.history.values.length === 0) continue;
  var data = d.history.timestamps.map(function(t, j) {
    return [t, d.history.values[j]];
  });
  series.push({
    name: d.name || ('DP ' + (i + 1)),
    type: 'line',
    data: data
  });
}
Important: Some ECharts examples use ES6 syntax like arrow functions (=>), let/const, or template literals. These work in modern browsers but for maximum compatibility, prefer var and function() syntax in the widget code editor.

Troubleshooting

Widget doesn't appear in the picker

  • Verify data/dashboard-wc/index.html exists in your project and contains the import map entry for wui-widget-custom-chart
  • Clear browser cache with Ctrl+Shift+R — the Dashboard uses a service worker that caches aggressively
  • Check that all 3 JSON files exist in data/WebUI/widgets-v2/StandardLibrary/Charts/

Widget appears but chart is blank

  • Open DevTools (F12) and check the Console for import errors
  • Navigate directly to https://<host>:<port>/dashboard/entry/custom-widgets.js — it should load without a 404
  • Ensure at least one datapoint is assigned in the widget settings

Chart code errors

  • JavaScript errors are shown in a red overlay at the bottom of the widget
  • Common issues: missing return statement, syntax errors, accessing undefined properties
  • Make sure your code returns a valid object, not undefined

History buffer is empty

  • The buffer only fills while the dashboard page is open — it starts empty on page load
  • Wait for datapoint values to change (the buffer stores each new value as it arrives)
  • Check that the datapoint is correctly configured and producing values

Changes not visible after editing files

  • Always hard-refresh with Ctrl+Shift+R
  • The Dashboard is a PWA — the service worker may serve cached versions of JavaScript files
  • In stubborn cases, open DevTools > Application > Service Workers > Unregister, then refresh

WinCC OA version upgrade

  • After upgrading WinCC OA, copy the new installation's index.html to your project
  • Re-add the import map entry for wui-widget-custom-chart
  • The widget files (custom-widgets.js, JSON schemas, SVG icon) are forward-compatible
